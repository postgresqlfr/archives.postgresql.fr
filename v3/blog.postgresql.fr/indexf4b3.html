Sortie de PostgreSQL-8.0.0

Par gleu le mercredi 19 janvier 2005, 09:08 - Dans les bacs - Lien permanent

Après plus d'une année de travail acharné, le groupe de développement PostgreSQL annonce fièrement la sortie de la version 8.0.0 du ORDBMS PostgreSQL. Cette version a demandé plus de modifications majeures que tout autre version jusqu'à la 6.3. Ces modifications incluent :

  • port Windows natif ;
  • points de sauvegardes/transactions imbriquées ;
  • gestion des exceptions à l'intérieur des fonctions ;
  • espaces logiques (tablespace) ;
  • PITR (Point in time recovery) ;
  • amélioration de la gestion des tampons partagés ;
  • processus d'écriture en tâche de fond pour réduire l'impact des points de vérification ;
  • délais configurables pour réduire l'impact de VACUUM ;
  • refonte de pg_dump et pg_restore ;
  • options étendues de formats de traces et rotation des traces ;
  • amélioration de la gestion des guillemets imbriqués pour les fonctions ;
  • ALTER TABLE peut modifier le type de données pour une colonne existante ;
  • amélioration des fonctionnalités d'ALTER pour tous les objets de la base de donnée.

Téléchargez-le maintenant :

  • miroirs FTP
  • version Windows
  • Bittorrent (inclut la version Windows)
  • Sourceforge (inclut la version Windows)

Certains paquets de distributions Linux populaires sont maintenant disponibles et plus le seront bientôt.

Les membres de la presse et le public non familier avec PostgreSQL devraient lire nos informations presse.

Les autres peuvent lire les notes de version pour une liste complète des modifications de la version 8.0, fonctionnalités et avertissements pour la mise à jour.

Téléchargez et amusez-vous !
