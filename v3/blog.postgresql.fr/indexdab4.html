PGWN - 17 juillet 2006

Par SAS le mercredi 19 juillet 2006, 15:11

Nouvelles hebdomadaires de PostgreSQL - 17 juillet 2006

Bruce Momjian et Tom Lane ont nettoyé les #includes de l'arbre des sources. Puis ils ont épuré le nettoyage.

Nous ne disposons pas à l'heure actuelle d'un nombre suffisant de machines sous Windows(R) dans la ferme de compilation. Si vous disposez d'un serveur sous Windows(R) qui peut faire tourner une compilation de PostgreSQL, au minimum toutes les nuits pour les tests, configurez-la et enregistrez-vous sur www.pgbuildfarm.org. Cela serait d'une grande aide pour faire évoluer PostgreSQL.

Josh Berkus purge actuellement /contrib. Il supprime les vieux projets qui ne satisfont pas les critères de contrib. La plupart sont déplacés vers pgFoundry, mais deux vont être supprimés : « tips » et mSQL-interface. Si quelqu'un souhaite reprendre la maintenance de ce code, qu'il se prononce rapidement, ou retrouve le code dans les archives FTP.

PGWN - 3 juillet 2006

Par Jean-Paul Argudo le mardi 4 juillet 2006, 08:40

Dans la version CVS TIP de PostgreSQL:

Greg Stark a ajouté une fonctionalité appelée INCLUDING CONSTRAINTS pour CREATE TABLE LIKE.

Christopher Kings-Lynne a ajouté une fonction PQescapeIdentifier() à la libpq.

Tom Lane a changé la syntaxe du constructeur ROW() afin que la liste des éléments de foo.* soit étendue à la liste des champs membres plutôt que de créer un champ de type rowtype imbriqué, comme c'était le cas avant. L'ancien comportement est toujours possible en oubliant le '.*'. On peut désormais faire un test de type IF ROW(NEW.*) IS DISTINCT FROM ROW(OLD.*) dans un trigger ou une rule.

Teodor Sigaev a changé l'algorithme de split pour GiST, comme il l'avait annoncé dans sa proposition.

Tom Lane a supprimé le processus de stats buffer et laisse les messages statistiques du backend aller directement au processus collector. Cela doit permettre une amélioration des performances.
